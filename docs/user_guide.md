## 시작하기

### 설치

1. Python 3.9 이상이 설치되어 있는지 확인하세요.
2. 프로젝트 폴더로 이동합니다.
3. 가상환경을 생성하고 활성화합니다:

```bash
python -m venv venv
source venv/bin/activate  # Windows: venv\Scripts\activate
```

4. 필요한 패키지를 설치합니다:

```bash
pip install -r requirements.txt
```

5. 애플리케이션을 실행합니다:

```bash
streamlit run app.py
```

### 웹 인터페이스

브라우저에서 `http://localhost:8501`로 접속하면 대시보드가 나타납니다.

## 옵션 계산기 사용법

### 1. 파라미터 입력

**기초자산 가격 (S)**
- 현재 주식이나 자산의 시장 가격
- 예: $100

**행사가격 (K)**
- 옵션 계약에서 약정된 가격
- 예: $105

**만기 (T)**
- 옵션이 만료되기까지 남은 일수
- 예: 30일

**무위험이자율 (r)**
- 연간 무위험 수익률 (%)
- 일반적으로 국채 수익률 사용
- 예: 5%

**변동성 (σ)**
- 기초자산의 연간 변동성 (%)
- 역사적 변동성 또는 내재 변동성 사용
- 예: 20%

**옵션 타입**
- Call: 살 권리
- Put: 팔 권리

### 2. 결과 해석

**옵션 가격**
- 콜/풋 옵션의 이론적 공정가치
- 시장 가격과 비교하여 저평가/고평가 판단

**내재가치**
- 즉시 행사 시 얻을 수 있는 이익
- Call: max(S - K, 0)
- Put: max(K - S, 0)

**시간가치**
- 프리미엄 - 내재가치
- 만기가 가까워질수록 감소

**Greeks**
- 옵션 가격의 민감도 지표
- 위험 관리에 활용

### 3. 손익 다이어그램

- X축: 기초자산 가격
- Y축: 옵션 보유자의 손익
- 녹색 선: 행사가격
- 빨간 선: 손익분기점

## 선물 계산기 사용법

### 1. 파라미터 입력

**현물 가격 (S)**
- 현재 시장 가격

**무위험이자율 (r)**
- 연간 무위험 수익률

**배당수익률 (q)**
- 연간 배당 수익률
- 주가지수의 경우 배당수익률 포함

**만기 (T)**
- 선물 계약 만료일까지 일수

### 2. 결과 해석

**이론적 선물가격**
- F = S × e^((r-q)T)
- 차익거래가 없는 경우의 공정가치

**베이시스**
- 선물가격 - 현물가격
- 정상시장(Contango): 베이시스 > 0
- 역조시장(Backwardation): 베이시스 < 0

## Greeks 분석

### Delta (Δ)

**의미**
- 기초자산 가격이 $1 변할 때 옵션 가격 변화
- 헷지 비율로 사용

**범위**
- 콜 옵션: 0 ~ 1
- 풋 옵션: -1 ~ 0

**해석**
- Delta 0.5: 기초자산 $1 상승 시 옵션 가격 $0.50 상승
- ATM 옵션의 Delta는 약 0.5

### Gamma (Γ)

**의미**
- Delta의 변화율
- Delta의 안정성 측정

**특징**
- 항상 양수
- ATM 옵션에서 최대
- 만기가 가까울수록 증가

**활용**
- Gamma가 큰 경우: Delta 헷지를 자주 조정 필요
- Gamma가 작은 경우: Delta 헷지가 안정적

### Theta (Θ)

**의미**
- 하루가 지날 때 옵션 가격 감소
- 시간 가치 감소율

**특징**
- 일반적으로 음수
- ATM 옵션에서 최대
- 만기가 가까울수록 증가

**활용**
- 옵션 매도자에게 유리 (시간 가치 수익)
- 옵션 매수자에게 불리 (시간 가치 손실)

### Vega (ν)

**의미**
- 변동성이 1% 변할 때 옵션 가격 변화

**특징**
- 항상 양수
- ATM 옵션에서 최대
- 만기가 길수록 증가

**활용**
- 변동성 증가 예상: 옵션 매수 (Long Vega)
- 변동성 감소 예상: 옵션 매도 (Short Vega)

### Rho (ρ)

**의미**
- 이자율이 1% 변할 때 옵션 가격 변화

**특징**
- 콜 옵션: 양수
- 풋 옵션: 음수

**활용**
- 일반적으로 다른 Greeks보다 영향 작음
- 장기 옵션에서 중요

## 고급 기능

### 복합 전략

여러 옵션을 조합하여 다양한 전략 구현:

**스트래들 (Straddle)**
- 같은 행사가격의 콜과 풋 동시 매수
- 큰 가격 변동 예상 시 사용

**스트랭글 (Strangle)**
- 다른 행사가격의 콜과 풋 매수
- 스트래들보다 낮은 비용

**스프레드 (Spread)**
- 같은 타입의 옵션을 사고 팔기
- 방향성 전략

### 헷징

**Delta 헷징**
- 옵션의 Delta만큼 현물 포지션 조정
- Delta-neutral 포지션 유지

**포트폴리오 헷징**
- 주가지수 선물로 포트폴리오 위험 관리
- 베타 조정

## 팁과 주의사항

### 팁

1. **변동성 추정**: 역사적 변동성과 내재 변동성을 함께 고려
2. **Greeks 모니터링**: 정기적으로 Greeks 확인하여 위험 관리
3. **시나리오 분석**: 다양한 파라미터로 시뮬레이션
4. **실시간 데이터**: 가능한 경우 실시간 시장 데이터 사용

### 주의사항

1. **모델의 가정**: Black-Scholes 모델은 여러 가정에 기반
   - 변동성 일정
   - 로그정규분포
   - 거래비용 없음

2. **시장 불완전성**: 실제 시장에서는
   - 거래비용 존재
   - 유동성 문제
   - 조기 행사 (아메리칸 옵션)

3. **교육용 도구**: 
   - 실제 투자 결정에 사용하지 말 것
   - 금융 전문가와 상담 필요

## 자주 묻는 질문 (FAQ)

**Q: 내재 변동성은 어떻게 구하나요?**

A: 시장에서 거래되는 옵션 가격을 Black-Scholes 모델에 역산하여 계산합니다.

**Q: 아메리칸 옵션도 계산할 수 있나요?**

A: 현재 버전은 유럽형 옵션만 지원합니다. 아메리칸 옵션은 이항 트리 모델이 필요합니다.

**Q: 실시간 데이터를 사용할 수 있나요?**

A: yfinance 패키지를 통해 Yahoo Finance 데이터를 불러올 수 있습니다.

**Q: 배당이 있는 경우는?**

A: 배당수익률(q)을 고려한 모델을 사용해야 합니다. 현재 선물 모델에서 지원합니다.

## 문제 해결

### 설치 오류

```bash
# 패키지 충돌 시
pip install --upgrade pip
pip install -r requirements.txt --force-reinstall
```

### 실행 오류

```bash
# 포트가 사용 중인 경우
streamlit run app.py --server.port 8502
```

### 계산 오류

- 파라미터가 유효한지 확인
- 만기가 0이 아닌지 확인
- 변동성이 너무 작지 않은지 확인
